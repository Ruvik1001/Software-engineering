# Лабораторные работы по курсу "Программная инженерия"

## Информация о проекте

Данный репозиторий содержит выполненные лабораторные работы по курсу "Программная инженерия". Проект представляет собой набор выполненных практических заданий, которые частично реализуют систему планирования задач на основе leadertask.ru, которая позволяет пользователям создавать цели и управлять задачами для их достижения.

## Описание системы

Система планирования задач предназначена для управления целями и задачами пользователей. Основные сущности системы:

- **Цель** - долгосрочная цель пользователя
- **Задача** - конкретная задача на пути к достижению цели
- **Исполнитель** - пользователь системы

### Функциональность

Система предоставляет следующий функционал:

- Создание нового пользователя
- Поиск пользователя по логину
- Поиск пользователя по маске имени и фамилии
- Создание новой цели
- Получение списка всех целей
- Создание новой задачи на пути к цели
- Получение всех задач цели
- Изменение статуса задачи в цели

## Структура проекта

Проект организован в виде отдельных директорий для каждой лабораторной работы:

```
.
├── lab_1/          # Документирование архитектуры в Structurizr
```

Каждая директория содержит:
- `system_design_task_XX.txt` - описание задания для соответствующей лабораторной работы
- Реализацию решения согласно требованиям задания

## Архитектура системы

### Общая архитектура

Система спроектирована с использованием современных подходов к разработке программного обеспечения:

1. **Многоуровневая архитектура** - разделение на слои представления, бизнес-логики и данных
2. **RESTful API** - для взаимодействия с клиентскими приложениями
3. **Микросервисная архитектура** (частично) - разделение функциональности на отдельные сервисы
4. **Event-Driven архитектура** - асинхронная обработка событий через брокер сообщений

### Компоненты системы

#### Lab 1: Документирование архитектуры
- Описание архитектуры системы с использованием Structurizr DSL
- Диаграммы System Context (C1) и Container (C2)
- Диаграммы Dynamic для ключевых сценариев использования

#### Lab 2: REST API сервис
- RESTful API для управления пользователями, целями и задачами
- Аутентификация и авторизация
- OpenAPI/Swagger документация
- Docker контейнеризация

#### Lab 3: Реляционная база данных
- Схема PostgreSQL для хранения данных
- Оптимизация запросов через индексы
- Миграции и тестовые данные

#### Lab 4: Документная база данных
- Схема MongoDB для альтернативного хранения данных
- Валидация схем документов
- CRUD операции и агрегации

#### Lab 5: Оптимизация производительности
- Кеширование часто запрашиваемых данных (Redis)
- Rate limiting для защиты API
- Стратегии инвалидации кеша

#### Lab 6: Event-Driven архитектура
- Проектирование событийной модели
- Интеграция с брокером сообщений (RabbitMQ/Kafka)
- Применение паттерна CQRS

## Технологический стек

- **Язык программирования**: Python
- **Веб-фреймворк**: FastAPI
- **Базы данных**: 
  - PostgreSQL (реляционная БД)
  - MongoDB (документная БД)
- **Кеширование**: Redis
- **Брокер сообщений**: RabbitMQ
- **Контейнеризация**: Docker, Docker Compose
- **Документирование архитектуры**: Structurizr DSL

## Запуск проекта

### Предварительные требования

- Docker и Docker Compose

### Быстрый старт

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Ruvik1001/Software-engineering.git
cd Software_engineering
```

2. Запустите необходимые сервисы через Docker Compose:

```bash
cd lab_1

```

3. Для каждой лабораторной работы следуйте инструкциям в соответствующей директории.
